# Documentation Decision Matrix

Version: 1.0

## Purpose
Standardize decisions about where new documentation belongs.

| Scenario | Document Type | Owner |
|---|---|---|
| Architectural decision | ADR | Architect |
| Technical specification | *_SPEC.md | Tech Lead |
| Development workflow | Playbook | Engineering |
| Coding convention | Standard | Engineering |
| Release evidence | Report | QA/Release |

## Rule
A document must have exactly one primary source of truth.
