# Purpose

This document defines the standard workflow for contributing to Tamer Studio.

The rules apply to:

- Developers
- AI Coding Assistants
- External Contributors

---

↓

↓

---

# Before Coding

Every contribution must:

✓ Understand PRODUCT

✓ Read PROJECT_CONTEXT

✓ Follow ARCHITECTURE

✓ Follow DEVELOPMENT

✓ Check ADR

✓ Review existing implementation

# Pull Request Checklist

- Architecture respected
- No duplicated logic
- Tests passed
- Documentation updated
- ADR updated (if needed)

---

# AI Contribution Rules

AI must:

Reuse before create.

Never ignore documentation.

Never bypass architecture.

Never invent requirements.

Always explain major decisions.
